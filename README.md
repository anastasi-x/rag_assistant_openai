# RAG Assistant OpenAI

Готовый RAG-ассистент на базе OpenAI API.

Проект реализует RAG-пайплайн: загружает документы из базы знаний, разбивает текст на чанки, создает embeddings, сохраняет данные в ChromaDB, ищет релевантный контекст по запросу пользователя и передает найденные фрагменты в OpenAI-модель для генерации ответа.

## Что умеет ассистент

* отвечает на вопросы по базе знаний;
* загружает несколько документов из папки `assistant_api/data/`;
* поддерживает форматы `.txt`, `.md`, `.csv`, `.html`, `.pdf`;
* использует OpenAI API для генерации ответов;
* использует ChromaDB как векторное хранилище;
* применяет `top_k=3` для выбора релевантных чанков;
* сохраняет повторяющиеся ответы в SQLite-кэш;
* показывает статистику по базе знаний и кэшу.

## Установка и запуск

Для стабильной работы проекта рекомендуется использовать Python 3.11.

1. Создать виртуальное окружение:

```bash
py -3.11 -m venv venv_py311
```

2. Активировать виртуальное окружение:

```bash
.\venv_py311\Scripts\Activate.ps1
```

3. Установить зависимости:

```bash
pip install -r requirements.txt
```

4. Создать файл `.env` по примеру `.env.example` и добавить OpenAI API ключ:

```env
OPENAI_API_KEY=your_openai_api_key_here
```

5. Добавить документы для базы знаний в папку:

```text
assistant_api/data/
```

Поддерживаемые форматы:

```text
.txt, .md, .csv, .html, .pdf
```

6. Перейти в папку ассистента:

```bash
cd assistant_api
```

7. Запустить проект:

```bash
python app.py
```

После запуска можно задавать вопросы в терминале.

## Команды в консоли

* обычный вопрос — ассистент ищет ответ в базе знаний;
* `stats` — показывает статистику системы;
* `clear` — очищает кэш;
* `quit`, `exit` или `q` — завершает работу.

## Обновление базы знаний

Если документы в папке `assistant_api/data/` были изменены, добавлены или удалены, нужно очистить локальную векторную базу и кэш, чтобы ассистент заново проиндексировал документы.

Для этого можно удалить локальные файлы и папки:

```text
assistant_api/chroma_db/
assistant_api/*.db
```

После этого нужно снова запустить:

```bash
python app.py
```

При запуске ассистент заново прочитает документы, разобьет их на чанки и добавит в ChromaDB.
